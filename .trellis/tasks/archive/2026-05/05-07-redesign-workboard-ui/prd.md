# 优化工作看板页面 UI — 信息架构、布局、视觉层级、色彩系统全面重构

## Goal

将 /briefing 页面从"装饰性深色大屏"重构为"清晰、紧凑、专业、高效率的工作看板"，从信息架构、布局比例、视觉层级、色彩系统四个维度全面优化。

## What I already know

- 核心组件：`DailyBriefingShell.tsx`（1012行）
- 样式文件：`index.css`（6684行，约393条 `.briefing-` 规则）
- 路由：`/briefing`
- 子组件：`BriefingTopPapers`、`BriefingProjectsSidebar`、`BriefingDateField`、`BriefingHistoryPicker`、`AutomationSettingsPanel`、`StatusBadge`、`UiIcon`
- 当前暗色变量：`--bg-page: #000000`, `--bg-panel: #1d1d1f`, `--bg-layer-1: #262628`, `--bg-layer-2: #28282a`
- 当前主色：`--accent-blue: #0071e3`

## Requirements

### 1. 页面命名统一
- 页面名称统一为"工作看板"，不再使用"研究看板 / 每日速览"
- 顶部 kicker 从"研究看板"改为"工作看板"
- 主标题从"每日速览"改为"今日工作概览"
- App.tsx 侧边栏链接文本从"每日动态和统计"改为可匹配的副标题

### 2. 顶部重构（压缩至 160-200px）
删除/合并：
- 删除大的"当前日报"卡片（日期、日历、历史按钮、4个指标卡）
- 删除大的"自动化设置"卡片
- 删除标题区重复的日期和状态文字
- 日期和统计信息不在顶部反复出现

新结构（5行紧凑布局）：
```
┌──────────────────────────────────────────────────────┐
│ 工作看板                              [手动] [时区]    │  ← 第1行
│                                                      │
│ 今日工作概览              [搜索框] [生成报告]         │  ← 第2行
│ 聚合今日论文、项目、风险与关键信号，用于快速判断处理优先级│
│                                                      │
│ 2026-05-07 · 最后生成 06:30 · 阅读进度 0% · 手动补充   │  ← 第3行
│ 论文候选 32 · 相关项目 5 · 订阅源 6 · 风险热点 6       │  ← 第4行
│                                                      │
│ 自动生成：每天 12:00 · Asia/Shanghai · 完成 06:32 [设置]│  ← 第5行
└──────────────────────────────────────────────────────┘
```

### 3. 三栏布局重新平衡
- 左侧目录：14%-18%（从 fixed 178px 改为更宽的相对宽度）
- 中间正文：52%-60%
- 右侧建议区：24%-30%
- 竖向摘要标记：移除，改为顶部横向筛选条或右侧卡片顶部分类标签

### 4. 左侧目录优化
- 加宽，避免文字截断
- 当前章节高亮更明显（更粗的指示线 + 更强的背景色）
- 一级/二级标题层级清晰（更好的缩进和字号区分）
- 可折叠结构（用 <details>/<summary> 或状态管理）
- 不要过度抢视觉焦点

### 5. 中间内容区优化
- 缩减"压迫感"：减小卡片内边距、段落间距
- 正文卡片不过宽过重
- 字体层级统一：主标题 → 模块标题 → 卡片标题 → 正文 → 辅助说明
- "今日重点"和"今日论文汇总"保持为核心内容

### 6. 右侧建议区优化
- 加宽，让英文标题有更舒适的换行
- 卡片内"建议动作 / 适合读者 / 为什么推荐"层级清晰
- 不过度堆叠彩色标签
- 统计数字用轻量样式（弱背景 + 细边框 + 小字号 + 小圆角）

### 7. 竖向摘要标记重设计
- 删除位于中间与右侧之间的 `.briefing-marker-rail`（竖向标记侧轨）
- 将摘要信息（建议/风险/参考数量）改为：
  - 顶部统计标签行的一部分（第4行）
  - 或右侧卡片顶部的轻量分类标签

### 8. 色彩系统统一
- 主色：蓝青色 `#0071e3`（保持）
- 深色背景层级重建：
  - `--bg-page`：页面背景（最深）
  - `--bg-panel`：区段背景
  - `--bg-layer-1`：卡片背景
  - `--bg-layer-2`：嵌套卡片/交互元素背景
  - 各层之间需肉眼可辨（当前 `#262628` vs `#28282a` 太接近）
- 强调色规则：
  - 蓝青（主操作、选中、链接）
  - 橙色（警告、风险）— 克制使用
  - 绿色（完成、成功）— 克制使用
  - 红色（危险、高风险）— 仅用于真正需要提醒处
- 普通标签：低饱和蓝灰（`color-mix(in srgb, var(--accent-blue) 18%, var(--bg-layer-1))`）
- 避免"很多地方都在亮"
- `box-shadow` / `glow` 只用于关键区域，不在全局滥用

### 9. 暗色背景层次重建
- 移除 `.briefing-command-deck` 的 `background: linear-gradient(90deg, ...)`
- 改用清晰的明度层级区分区域
- 卡片背景比页面背景略亮（通过新 `--bg-card` 等变量）
- 分割线和边框要轻（`--border-subtle: rgba(255,255,255,0.05)` 或更低）
- 阴影只用于悬浮卡片和弹窗

### 10. 字体和卡片权重
- 页面名称：11px / uppercase / muted
- 页面主标题：28px / bold（从 38px 减小）
- 模块标题：18px / bold
- 卡片标题：14-16px / medium-bold
- 正文：14px / regular
- 辅助说明：12px / muted
- 所有模块不再同时"强调"

## Technical Approach

1. **先改 CSS**：新增/修改暗色变量定义，重建色彩层级
2. **重构顶部 JSX**：重写 `<header>` 部分的 JSX 结构
3. **调整三栏布局**：修改 `.briefing-grid` 和相关 CSS Grid 定义
4. **优化子组件样式**：统一卡片、标签、按钮、边框样式
5. **移除竖向标记**：删除 `.briefing-marker-rail` 相关 JSX 和 CSS
6. **不要大改业务逻辑**：保持 data fetching、state management、事件处理不变

## Decision (ADR-lite)

- **Context**: 当前 /briefing 页面顶部过高、信息重复、暗色层级糊、颜色杂、三栏比例失衡
- **Decision**: 全面重构 DailyBriefingShell 的 JSX 结构和 CSS 样式，保持数据流和业务逻辑不变
- **Marker Rail**: 直接移除 `.briefing-marker-rail` 完整模块（JSX + CSS）。将建议/风险/参考计数合并到顶部状态摘要栏作为轻量统计标签。如筛选功能需要，在右侧建议区顶部添加横向筛选标签（低饱和蓝灰背景，选中项用蓝青高亮）。不使用多色点或浮动竖条。
- **Consequences**: 需同时修改 TSX 和 CSS，涉及面广但每处改动明确；通过分 step 导航到具体模块确保完整性
